Database Design

MongoDB Schema with Relationships

User Model

  • name: String (required)
  • email: String (unique)
  • phone: String
  • role: buyer/seller/broker/admin
  • subscriptionStatus: active/expired
  • subscriptionExpiry: Date
  • avatar: String (URL)
  • isVerified: Boolean

Property Model

  • title: String (required)
  • description: String
  • type: land/house/rental/commercial
  • price: Number
  • location: Object (district, parish)
  • features: Object (beds, baths, size)
  • images: Array of URLs
  • owner: ObjectId (ref: User)
  • status: draft/active/sold

Chat Model

  • participants: [ObjectId] (ref: User)
  • property: ObjectId (ref: Property)
  • lastMessage: ObjectId (ref: Message)
  • unreadCount: Number
  • isActive: Boolean
  • createdAt: Date
  • updatedAt: Date

Message Model

  • chat: ObjectId (ref: Chat)
  • sender: ObjectId (ref: User)
  • content: String
  • messageType: text/image/file
  • isRead: Boolean
  • readAt: Date
  • createdAt: Date

Notification Model

  • user: ObjectId (ref: User)
  • type: String (enum)
  • message: String
  • data: Object (additional info)
  • status: pending/sent/failed
  • channels: [email, sms, whatsapp]
  • sentAt: Date
  • isRead: Boolean

Subscription Model

  • user: ObjectId (ref: User)
  • plan: free/premium/enterprise
  • amount: Number
  • status: active/expired/cancelled
  • startDate: Date
  • endDate: Date
  • paymentMethod: String
  • reference: String

Database Relationships

User Properties (1:Many) Chats (Many:Many) Messages (1:Many)
